# 2026-07-08_김용성_프로_HPM-HPC-POS_정산_구조_설명

HPM(해피페이머니)–HPC(해피포인트 시스템)–POS–ERP 간 정산 데이터 흐름을 김용성 프로가 설명. HPP 결제 시 유상/무상 분리 요건 때문에 유상 전환분이 HP 사용 실적에서 상계(0) 처리되는 로직으로 구현되어 있어, 기대했던 처리 방식(사용 전액 집계)과 다름이 확인됨.

## 대화 정보
- **참여자:** 김용성 프로, Ellie Hyun
- **채널:** 사내 메신저
- **일시:** 2026-07-08

## 📌 주요 내용
- **AS-IS:** POS는 적립 데이터를 별도로 남기지 않아 ERP로 직접 데이터를 넘길 수 없고, HPC(해피포인트 시스템)가 ERP로 데이터를 전달해 왔음.
- **TO-BE (HPP):** HPM ↔ HPC ↔ POS 구간으로 자금(포인트)이 오가야 하는 구조.
- **사례:** HPM → HPC 20P 전환 + HPC → POS 120P 사용인 경우, 사업팀은 HPC에서 "HP 사용 120P"로 처리될 것으로 기대함.
- **실제 구현:** 희경 프로가 최초 받은 요건상 유상/무상 포인트가 섞이면 안 되어, HPM → HPC 전환분을 +20 −20 = 0으로 상계 처리하고 HPC → POS는 "무상 100P 사용"으로만 처리하는 로직으로 구현해 둠.
- 이에 따라 유상 전환분 20P가 HP 사용 집계에 잡히지 않는 구조 → 정산·집계 관점의 정합성 확인 필요.
